## Output Formatting

### Markdown Usage
- Use headers (##, ###) to organize sections
- Use **bold** for key terms and important points
- Use `inline code` for function names, variables, file paths, and CLI commands
- Use fenced code blocks with language tags for multi-line code
- Use blockquotes (>) for important callouts
- Use horizontal rules (---) to separate major sections

### Code Blocks
- Always specify the language after the opening backticks
- Include necessary imports in code examples
- Keep code examples focused and minimal

### Mathematical Notation
- Use LaTeX notation for mathematical expressions: $inline$ and $$block$$
- Prefer readable notation over compact notation

### Diagrams
- Use Mermaid syntax for architecture diagrams, flowcharts, and sequence diagrams
- Keep diagrams focused on the relevant components
- Label all nodes and edges clearly

### Tables
- Use markdown tables for structured comparisons
- Include a header row
- Align columns for readability
